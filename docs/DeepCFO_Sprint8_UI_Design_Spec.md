## 1. 产品定位与设计愿景

### 1.1 产品概述

| 属性 | 说明 |
|------|------|
| 产品名称 | DeepCFO — AI 财税助手 |
| 产品类型 | B2B SaaS（面向小微企业的智能财税工具） |
| 目标用户 | 小微企业主（1-20人）、创业者、个体工商户 |
| 核心场景 | 移动端优先（>70%流量来自手机），桌面端补充办公 |
| 核心价值 | 智能财税自动化 — 让不懂财务的老板也能轻松管账、报税、分析经营状况 |

### 1.2 设计愿景

**"财税管理，从令人头疼变成令人安心。"**

DeepCFO 的目标是用现代互联网产品的体验，重新定义小微企业的财税工具。传统财税软件普遍存在以下痛点：界面沉闷（深色边框+密密麻麻的表格）、操作复杂（需要专业知识才能上手）、反馈不及时（看不到经营全景）。我们希望通过以下设计哲学来解决这些问题：

- **清爽不压抑**：大量留白，充足呼吸感，信息密度适中。用户打开应用不会感到"信息过载"或"压力山大"。
- **专业但友好**：色彩有温度（主色调选用偏暖的蓝色而非纯冷蓝），图形化代替表格化，降低认知门槛。
- **高效不冗余**：常用操作一步直达，次要信息优雅折叠。用户不需要思考"这个功能在哪"。
- **可信赖**：财税数据敏感，视觉语言要传达"稳定、可靠、准确"。不过度花哨，也不过度沉闷。

### 1.3 设计关键词

```
清爽 · 专业 · 值得信赖 · 轻量感 · 高效 · 温暖 · 智能
```

### 1.4 设计原则优先级

1. **清晰 > 美观**：所有设计决策优先保证信息传达无误，在此基础上追求美感。
2. **一致 > 创新**：全局组件保持视觉一致性，不为单个页面的"亮点"破坏整体节奏。
3. **克制 > 丰富**：动效、装饰、色彩都应克制使用。宁可少一个特效，不要多一分噪音。
4. **移动优先 > 桌面优先**：所有设计优先满足手机端体验，大屏端做合理扩展。

## 2. 色彩系统

### 2.1 设计理念

色彩是产品个性的第一表达。DeepCFO 的色彩体系围绕"可信赖的科技感 + 温暖的亲和力"展开：

- **蓝色系**作为主色，传递"稳定、专业、科技"，但选用了偏暖的 `#1A53C8` 而非纯正的科技蓝，让视觉感受更温和。
- **绿色系**代表"正向、增长、成功"，用于收入类数据和完成状态，给用户正向反馈。
- **橙色系**代表"待处理、中度紧急"，黄色警示但不制造焦虑。
- **红色系**保留给真正需要关注的"逾期、错误"，不滥用。

### 2.2 主色（Primary）

| 色值名称 | HEX | RGB | 使用场景 |
|---------|-----|-----|---------|
| 主蓝（Deep Blue） | `#1A53C8` | rgb(26, 83, 200) | 主按钮背景、主要操作、激活态图标和文字 |
| 浅蓝（Light Blue） | `#4F7DF3` | rgb(79, 125, 243) | 渐变过渡色、次级强调、AI按钮渐变、图表主色 |
| 极浅蓝（Pale Blue） | `#D6E4FF` | rgb(214, 228, 255) | 激活Tab背景、hover态背景、选中态背景 |

> **为什么选择这个蓝色？** 纯正科技蓝（#007AFF / #2962FF）偏冷，适合纯工具类产品。DeepCFO 作为面向非专业老板的产品，选用 `#1A53C8` 带有微妙的暖调，视觉上更"柔和可亲"，减少财税软件给用户带来的距离感和压迫感。

### 2.3 正向色（Success / Positive）

| 色值名称 | HEX | RGB | 使用场景 |
|---------|-----|-----|---------|
| 翠绿（Emerald） | `#10B981` | rgb(16, 185, 129) | 收入数字、增长指标、完成状态图标 |
| 深绿（Deep Emerald） | `#006947` | rgb(0, 105, 71) | 正向徽章文字、强调正向 |
| 浅绿（Light Emerald） | `#D1FAE5` | rgb(209, 250, 229) | 正向徽章背景、成功状态背景 |

### 2.4 警示色（Warning / Caution）

| 色值名称 | HEX | RGB | 使用场景 |
|---------|-----|-----|---------|
| 琥珀橙（Amber） | `#F59E0B` | rgb(245, 158, 11) | 警示图标、进行中状态 |
| 深橙（Deep Amber） | `#92400E` | rgb(146, 64, 14) | 警告徽章文字 |
| 浅橙（Light Amber） | `#FEF3C7` | rgb(254, 243, 199) | 警告徽章背景、待处理行背景 |

### 2.5 危险色（Danger / Error）

| 色值名称 | HEX | RGB | 使用场景 |
|---------|-----|-----|---------|
| 红色（Red） | `#EF4444` | rgb(239, 68, 68) | 危险按钮背景、错误图标、逾期文字 |
| 深红（Deep Red） | `#991B1B` | rgb(153, 27, 27) | 危险徽章文字 |
| 浅红（Light Red） | `#FEE2E2` | rgb(254, 226, 226) | 危险徽章背景、错误状态背景 |

> **重要提示**：红色在财税软件中使用场景极少（只有真正的错误、逾期、删除确认时才用）。如果某个"待处理"事项被设计成红色，用户会产生不必要的焦虑。**严格区分"警告"（待处理，橙色）和"危险"（逾期/错误，红色）的使用场景。**

### 2.9 色彩语义速查表

| 语义 | 主色 | 背景 | 文字 | 典型场景 |
|------|------|------|------|---------|
| 主要操作 | `#1A53C8` | — | `#FFFFFF` | 主按钮 |
| 次要操作 | `#F4F6FF` | `#F4F6FF` | `#1A53C8` | 次按钮 |
| 收入/正向 | `#10B981` | `#D1FAE5` | `#006947` | 收入数字、正向徽章 |
| 待处理/警告 | `#F59E0B` | `#FEF3C7` | `#92400E` | 待申报、进行中 |
| 逾期/危险 | `#EF4444` | `#FEE2E2` | `#991B1B` | 逾期提示、错误提示 |
| 草稿/中性 | `#64748B` | `#F1F5F9` | `#64748B` | 草稿状态 |

### 2.10 色彩使用禁忌

1. **禁止滥用红色**：`#EF4444` 只用于错误和删除确认，禁止用于普通"待处理"状态。
2. **禁止主背景使用主色**：主色面积过大（整块背景）会降低可读性和专业感。
3. **禁止背景渐变**：页面和卡片背景不使用渐变，保持纯色背景的清爽感。
4. **禁止4色以上同框**：一个卡片或模块内同时出现的强调色不超过2种。
5. **正向色用于增长，警示色用于状态**：不要混用（例如：亏损时用红色而非绿色）。

## 3. 字体系统

### 3.1 字体堆叠（Font Stack）

```css
/* 标题字体 */
font-family: 'Manrope', 'Noto Sans JP', 'Noto Sans KR', 'Noto Sans SC', system-ui, -apple-system, sans-serif;

/* 正文字体 */
font-family: 'DM Sans', 'Noto Sans JP', 'Noto Sans KR', 'Noto Sans SC', -apple-system, BlinkMacSystemFont, sans-serif;
```

> **为什么不用 Inter？** Inter 是最常见的无衬线字体，因此"没有个性"。在竞品中脱颖而出，需要一个有辨识度但不失专业感的字体。DM Sans 的圆润终端和 Manrope 的几何感形成 DeepCFO 独特的气质。

### 3.2 字号层级（Type Scale）

基于 4px 基准网格：

| 级别名称 | 字号 | 字重 | 行高 | 使用场景 |
|---------|------|------|------|---------|
| 大数字（KPI） | 32px | 800 (ExtraBold) | 1.2 | 仪表盘 KPI 数字、月度金额 |
| 页面标题（H1） | 28px | 700 (Bold) | 1.3 | 页面大标题 |
| 卡片标题（H2） | 18px | 700 (Bold) | 1.4 | 卡片标题、模块标题 |
| 大正文（H3） | 16px | 600 (Semibold) | 1.5 | 表格表头、重要正文 |
| 正文（P） | 14px | 400 (Regular) | 1.6 | 主体内容、说明文字 |
| 辅助文字 | 12px | 500 (Medium) | 1.5 | 时间戳、辅助说明 |
| 标签/徽章 | 11px | 600 (Semibold) | 1.2 | 状态徽章、小标签 |
| 导航文字 | 10px | 600 (Semibold) | 1.0 | 底部导航标签 |

### 3.3 数字字体规范

```css
/* 数字统一使用 Manrope ExtraBold */
font-family: 'Manrope', 'Noto Sans SC', system-ui, sans-serif;
font-weight: 800;
font-variant-numeric: tabular-nums; /* 等宽数字，上下对齐 */
```

**等宽数字的重要性**：在财务报表和对账场景中，用户经常需要目测对比上下行数字。`tabular-nums` 属性强制所有数字等宽，无论数值大小，数字始终占据相同宽度，大幅提升阅读效率。

## 6. 组件规范

## 8. 动效规范

### 8.1 动效设计理念

动效是"无声的交互语言"。DeepCFO 的动效遵循以下原则：

- **有目的**：每个动效都有功能意义——引导注意力、确认操作、表达状态变化，而非装饰。
- **克制但有存在感**：动效不应抢夺内容注意力，但也不应完全没有。
- **快**：移动端用户期望即时反馈。动效拖慢操作会被用户感知为"卡顿"。
- **弹性**：轻微的弹性（spring）动效比线性动画更自然、更现代。

### 8.2 时长规范（Duration）

| 动效类型 | 时长 | 缓动函数 | 使用场景 |
|---------|------|---------|---------|
| 即时反馈 | 0.15s | ease-out | 点击反馈、hover变色 |
| 快速过渡 | 0.2s | ease | 按钮状态切换、徽章变色 |
| 标准动画 | 0.3s | ease-out | 页面进入、卡片展开、Tab切换 |
| 强调动画 | 0.5s | ease-in-out | 模态框出现、重要状态变化 |
| 持续动画 | 循环 | ease-in-out | AI按钮浮动、加载动画 |

### 8.3 缓动函数（Easing）

```css
/* 标准缓动（大多数场景） */
transition-timing-function: ease; /* cubic-bezier(0.4, 0, 0.2, 1) */

/* 进入缓动（元素出现时） */
transition-timing-function: ease-out; /* cubic-bezier(0, 0, 0.2, 1) */

/* 退出缓动（元素消失时） */
transition-timing-function: ease-in; /* cubic-bezier(0.4, 0, 1, 1) */

/* 往复缓动（打开/关闭循环） */
transition-timing-function: ease-in-out; /* cubic-bezier(0.4, 0, 0.2, 1) */

/* 弹性动效（按钮点击反馈） */
transition-timing-function: cubic-bezier(0.34, 1.56, 0.64, 1); /* Spring-like */
```

### 8.4 常见动效场景

#### 点击反馈动效

```css

  transition: transform 0.15s cubic-bezier(0.34, 1.56, 0.64, 1);
}

  transform: scale(0.97);
}
button:hover {
  transform: scale(1.02);
}
```

```css
    opacity: 0;
    transform: translateY(8px);

  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card:nth-child(2) { animation-delay: 50ms; }
.card:nth-child(3) { animation-delay: 100ms; }
.card:nth-child(4) { animation-delay: 150ms; }
```

```css

  overflow: hidden;

```

#### AI 浮动按钮动画

### 8.5 动效禁用场景

以下场景应减少或禁用动效：
- **用户设置了"减少动画"（prefers-reduced-motion）**：尊重系统设置
- **低性能设备**：通过 DeviceMemory API 检测，低端设备降级动画
- **纯数据展示列表**：大表格（>50行）不添加交错入场动画
- **打印模式**：打印时不显示任何动画效果

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

- 变化趋势用百分比 + 箭头表示，数字较小（不抢主体数字风头）

**待办预警区**：
- 按紧急程度排序：红色（逾期/3天内）> 橙色（7天内）> 蓝色（提示）
- 红色不用于"已完成"或"草稿"状态，严格区分语义
- 每条只显示核心信息（一句话说清楚），详细进入详情页

**快捷操作区**：
- 6个最常用操作，2行×3列网格
- 图标 + 文字双要素，图标传递视觉记忆，文字消除歧义
- 图标使用实心图标（如 `add_circle`）而非线图标，增加触摸目标感

**AI 建议卡片**：
- AI 建议不是首页的"装饰"，而是真正有价值的洞察
- 建议内容由 AI 模型生成，展示业务逻辑（如"您3月的差旅费比2月高15%"）
- 点击后进入 AI 对话界面，获取详细分析

## 11. 响应式策略

### 11.1 响应式断点与布局映射

| 断点 | 宽度 | 导航方式 | 内容列数 | 主要布局变化 |
|------|------|---------|---------|------------|
| xs | < 640px | 底部Tab | 1列 | 卡片全宽，KPI 双列 |
| sm | 640-767px | 底部Tab | 1-2列 | KPI 双列 |
| md | 768-1023px | 底部Tab | 2列 | 开始出现侧边栏（可收起） |
| lg | 1024-1279px | 侧边栏 | 2-3列 | Bento Grid 2行 |
| xl | ≥ 1280px | 侧边栏 | 3-4列 | 完整 Bento Grid |

### 11.2 移动端适配原则

1. **触摸优先**：所有可点击元素的最小触摸区域 ≥ 44px × 44px
2. **拇指区**：核心操作放在屏幕下半部分（拇指易触达区域）
3. **单手操作**：主要流程保证单手可完成，不强制双手操作
4. **滑动替代点击**：长列表优先使用滑动操作（swipe actions）而非展开按钮
5. **表单简化**：移动端表单字段尽量单列，减少横向滚动

### 11.3 桌面端适配原则

1. **鼠标 hover**：所有可点击元素添加 hover 态（移动端无 hover）
2. **右键菜单**：重要操作支持右键菜单（如凭证列表的编辑/删除/导出）
3. **键盘导航**：Tab 键可完整导航，Enter 键确认操作
4. **快捷键**：常用操作提供键盘快捷键（如 Ctrl+S 保存，Ctrl+N 新增）
5. **内容密度**：桌面端可适当提高信息密度，但不超过移动端的 2 倍

## 12. Accessibility无障碍规范

### 12.1 色彩对比度

| 元素组合 | 最低对比度 | 目标对比度 | 状态 |
|---------|-----------|-----------|------|
| 主按钮文字/背景 | 4.5:1 | 5.5:1 | ✅ |
| 正文文字/页面背景 | 4.5:1 | 6:1 | ✅ |
| 辅助文字/页面背景 | 3:1 | 4:1 | ✅ |
| 图表文字/背景 | 3:1 | 4:1 | ✅ |
| 纯装饰性元素 | 无要求 | — | — |

### 12.2 ARIA 标签规范

```html
<!-- 按钮 -->
<button aria-label="新增发票">
  <span class="material-symbols-outlined">add</span>
</button>

<!-- 图标按钮（无文字） -->
<button class="icon-btn" aria-label="删除此凭证">
  <span class="material-symbols-outlined" aria-hidden="true">delete</span>
</button>

<!-- 徽章状态 -->
<span class="badge badge-success" role="status" aria-label="状态：已完成">
  已完成
</span>

<!-- 进度指示 -->
<div role="progressbar" aria-valuenow="50" aria-valuemin="0" aria-valuemax="100" aria-label="申报进度">
  50%
</div>

<!-- AI 浮动按钮 -->
<button class="ai-fab" aria-label="打开 AI 财税助手">
  <span class="material-symbols-outlined" aria-hidden="true">psychology</span>
</button>
```

### 12.3 焦点管理

- 所有交互元素（按钮、输入框、链接）都有可见焦点样式
- 焦点样式：`outline: 2px solid #1A53C8; outline-offset: 2px`（不使用默认浏览器蓝框）
- 模态框打开时焦点锁定在模态框内，关闭后焦点回到触发元素
- 页面加载时焦点不自动跳转，避免屏幕阅读器用户被打断
